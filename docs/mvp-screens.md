## Екран 01 — Welcome / Онбординг
*Перший запуск · 7 кроків · 3–4 хвилини*

**Призначення:** Пояснити суть додатку, зібрати мінімальний профіль і запустити першу рекомендацію. Виконується один раз. Далі — пропускається.

### Стартовий splash-екран

| Поле / елемент | Тип | Опис |
|---------------|-----|------|
| Лого + назва | Статика | «Каспер. Крок за кроком.» на темно-зеленому фоні |
| Підзаголовок | Текст | «AI-супутник для поступового повернення у форму» |
| Лапка / кіт | Зображення | Реалістичний образ Каспера або відбиток лапки |
| Слоган | Текст | «Не рекорд. Ритм, що тримає.» |
| CTA | Кнопка | «Почати →» — веде до кроку 1 |

### 7 кроків онбордингу

| Крок | Зміст |
|------|-------|
| **1 — Профіль** | Хто тренується? Серж / Олена / Новий профіль. Визначає режим AI. |
| **2 — Ім'я і вік** | Ім'я (обов'язково). Вікова група: 20–30 / 31–40 / 41–50 / 51–60 / 60+. Стать (опційно). |
| **3 — Стан** | Рівень активності зараз (4 варіанти). Коли востаннє тренувався (4 варіанти). |
| **4 — Здоров'я** | Чи є обмеження? Чеклист: операція / серце / суглоби / хронічний біль / немає. Вільне поле «заборони лікаря». Дата події. |
| **5 — Цілі** | Мультивибір: повернути ритм / відновлення / витривалість / підготовка до бігу / разом з партнером / самопочуття. |
| **6 — Розклад** | Час: 10–15 / 20–30 / 30–45 / по-різному. Час доби. К-сть тренувань: 1–2 / 3–4 / 5+ / розберемось. |
| **7 — Партнер** | Чи тренуєшся з кимось? Посилання / QR-код / пізніше. Пояснення режиму «Разом». |

> Принцип: відповіді — відправна точка. Каспер адаптується в процесі. Будь-яке поле змінюється в налаштуваннях.

**Навігація після онбордингу:**
- Крок 7 завершено → Екран 02 → Екран 03
- Повторний запуск → одразу Екран 02

## Екран 02 — Вибір профілю
*Кожен запуск · перемикання між профілями · вибір режиму*

**Призначення:** Дозволяє обрати, хто зараз тренується.

### Елементи екрану

| Поле / елемент | Тип | Опис |
|---------------|-----|------|
| Картка «Серж» | Кнопка-картка | Ім'я, аватар, поточна серія, статус дня |
| Картка «Олена» | Кнопка-картка | Ім'я, аватар, серія, статус. Якщо є обмеження — значок «режим відновлення» |
| Кнопка «Разом» | CTA-кнопка | Запускає спільну сесію. Показує кількість спільних прогулянок тижня. |
| Дата / день тижня | Статика | Відображається зверху для контексту |
| Нагадування | Банер (умовно) | Якщо є невиконане нагадування — м'яка підказка |

### Логіка режиму «Разом»
- Один ініціює → інший отримує push-нагадування і підтверджує
- Кожен заповнює pre-check незалежно (не бачать відповідей одне одного)
- AI формує два персональні плани і спільний план прогулянки
- Post-check — кожен заповнює окремо. Вікно 2 год після завершення
- Якщо один не заповнив — нагадування через год; через дві — запис з позначкою «неповні дані»

## Екран 03 — Головна (дашборд)
*Щодня після вибору профілю · центральний екран*

**Призначення:** «Що нам сьогодні робити безпечно і корисно?»

### Блоки екрану (зверху вниз)

| Поле / елемент | Тип | Опис |
|---------------|-----|------|
| Шапка | Статика | Ім'я профілю, дата, режим. Іконка налаштувань. |
| **Рекомендація дня** | AI-картка | Тривалість, темп, режим обережності. Причина (1 речення). |
| Кнопка «Почати» | CTA | Головна дія → pre-check. Великий, помаранчевий. |
| Кнопка «День відновлення» | Вторинна CTA | З'являється якщо рекомендований відпочинок або червоний прапорець. |
| Тиждень — статистика | Мінікарти | Хвилин / Прогулянок / Серія (streak) / Спільний час |
| Самопочуття-тренд | Мікрографік | Середня оцінка post-check за 7 днів. Тенденція. |
| Маленька ціль тижня | Картка | Одна конкретна ціль від Каспера. |
| Фраза Каспера | Текст | М'яка фраза підтримки. Оновлюється щодня. |

### Умовна логіка рекомендації

| Стан | Дія |
|------|-----|
| Самопочуття нормальне | Планова ходьба або м'який прогрес |
| Легка втома | Коротший або комфортніший варіант |
| Дискомфорт / жовтий | Зменшити навантаження, не підвищувати |
| Червоний прапорець | Не рекомендувати активність. «Сьогодні відпочиваємо.» |

> ⚠️ Для Олени: рекомендація завжди враховує медичні обмеження ПЕРШ НІЖ формувати текст. Якщо є «не бігти» — біг ніколи не пропонується.

## Екран 04 — Активність
*Pre-check · Фіксація · Post-check · Три під-екрани однієї сесії*

### Під-екран 4А — Pre-check (до активності)

| Поле / елемент | Тип | Опис |
|---------------|-----|------|
| Сон | Слайдер 1–5 | Як спалось? |
| Втома | Слайдер 1–5 | Рівень загальної втоми |
| Біль / дискомфорт | Так / Ні | Якщо «Так» → поле деталей і зона тіла |
| Готовність | Шкала 1–5 | Чи готово тіло рухатись |
| Настрій | Emoji-вибір | 5 варіантів: дуже погано → відмінно |
| Умови | Опційно | Надворі / вдома / тренажерний зал |
| **Для Олени (окремо)** | Банер | «Чи є дискомфорт у зоні операції сьогодні?» — ПЕРШЕ питання |
| CTA «Поїхали» | Кнопка | Підтверджує готовність. Запускає таймер. |

### Під-екран 4Б — Сесія (під час ходьби)

| Поле / елемент | Тип | Опис |
|---------------|-----|------|
| Таймер | Активний | Відраховує час. Кнопки: пауза / завершити. |
| Цільовий час | Статика | Показує рекомендований час |
| Нотатка | Текст (опц.) | Вільне поле для коментаря |
| Підказка Каспера | Банер (опц.) | Наприклад: «Якщо залишилось 3 хв — спробуй прискоритись» |
| **Для Олени** | Банер | «Комфортний темп. Без прискорення. Фокус — рівне дихання.» |
| «Завершити» | CTA | Зупиняє таймер → post-check |

### Під-екран 4В — Post-check (після активності)

| Поле / елемент | Тип | Опис |
|---------------|-----|------|
| Тривалість | Авто / ручна | Час з таймера або ручне введення |
| Дистанція | Число | Км (опційно) |
| Кроки | Число | Опційно. Вручну або з Garmin. |
| Пульс (середній) | Число | Опційно. Для Олени важливо — вручну з Garmin. |
| Складність | Шкала 1–10 | 1 = дуже легко, 10 = дуже важко |
| Втома після | Шкала 1–5 | Наскільки втомлено після? |
| Дискомфорт | Так / Ні | Якщо «Так» → деталі. Для Олени: окреме про зону операції. |
| Самопочуття | Краще / Так само / Гірше | Порівняно з тим, як було до |
| Що змінити | Текст (опц.) | «Наступного разу…» |
| Коментар | Текст (опц.) | Загальний коментар |
| CTA «Зберегти» | Кнопка | Зберігає. Показує висновок Каспера. |

> Після збереження Каспер показує картку: підсумок сесії + висновок (🟢/🟡/🔴) + рекомендація на завтра.

## Екран 05 — Журнал
*Всі записи активностей · фільтр · деталі*

### Елементи екрану

| Поле / елемент | Тип | Опис |
|---------------|-----|------|
| Фільтр профілю | Таби | Серж / Олена / Разом / Всі. Зберігає вибір. |
| Список карток | Скрол-список | Зворотна хронологія. Найновіше — зверху. |
| Порожній стан | Ілюстрація | Кіт зі слідами і фраза «Перший крок ще попереду» |

### Картка активності в списку

| Поле | Опис |
|------|------|
| Дата і день тижня | «Вівторок, 14 травня» |
| Режим | Серж / Олена / Разом |
| Тривалість | «22 хв» |
| Складність | 🟢 ≤4 / 🟡 5–7 / 🔴 ≥8 |
| Самопочуття-після | Краще ↑ / Так само → / Гірше ↓ |
| Значок прапорця | З'являється якщо зафіксовано дискомфорт або червоний прапорець |

### Екран деталей запису

- Вся інформація сесії (дата, тривалість, режим, дистанція, кроки, темп, пульс, нотатка)
- Pre-check дані (сон, втома, готовність, настрій, умови)
- Post-check дані (складність, втома, дискомфорт, що змінити, коментар)
- Висновок Каспера (збережений 🟢/🟡/🔴)
- «Редагувати» — виправити дані вручну

## Екран 06 — Статистика
*Персональний і спільний прогрес · тижневий підсумок · тренди*

### Перемикач перегляду

| Вкладка | Зміст |
|---------|-------|
| **Серж** | Особистий прогрес: рух вперед, готовність до прогресу |
| **Олена** | Особистий прогрес: стабільність відновлення, маркери дискомфорту |
| **Разом** | Тільки спільне: час разом, серія, регулярність |

### Персональна статистика

| Блок | Опис |
|------|------|
| KPI цього тижня | Хвилин / Прогулянок / Серія / Середня складність |
| Графік по днях | Активний день / відновлення / пропуск. 7 або 30 днів. |
| Тренд самопочуття | Середнє pre і post-check за тиждень. |
| Прогрес до цілі | Серж: хід до швидкої ходьби. Олена: стабільність без погіршень N тижнів. |
| Тижневий підсумок | Текст Каспера: підсумок, спостереження, фокус наступного тижня. |
| Маркери відновлення (Олена) | Дні з дискомфортом, динаміка зони операції |

### Спільна статистика (Разом)

| Блок | Опис |
|------|------|
| Спільних прогулянок | Цього тижня / за весь час |
| Час разом | Хвилини і години |
| Серія (streak) | Найдовша і поточна спільна серія |
| Регулярність тижнів | Кількість тижнів з хоча б однією спільною прогулянкою |
| Найкращий спільний тиждень | Коли обидва показали стабільний результат |
| Карта тижнів | Сітка: кожна клітинка = тиждень |

> ⚠️ Темп, дистанція і складність — виключно в персональній статистиці. Ніколи не порівнюються між профілями.

## Екран 07 — Налаштування
*Профіль · Здоров'я та обмеження · Нагадування · Система*

### Розділ A — Базовий профіль

| Поле | Тип | Опис |
|------|-----|------|
| Ім'я | Текст | Редагується |
| Вікова група | Вибір | 20–30 / 31–40 / 41–50 / 51–60 / 60+ |
| Стать | Вибір (опц.) | Для розрахунку пульсових зон |
| Поточний рівень | Вибір | 4 варіанти активності |
| Цілі | Мультивибір | Такі ж як в онбордингу. Редагуються. |
| Партнер | Статус / запрос | Хто пов'язаний в режимі «Разом» |

### Розділ Б — Здоров'я та обмеження (захищений)

| Поле | Тип | Опис |
|------|-----|------|
| Статус відновлення | Toggle | «Зараз у процесі відновлення» → підвищена обережність |
| Вид обмеження | Чеклист | Операція/травма / серце/тиск / суглоби / хронічний біль / вагітність / немає |
| Дата події | Дата | Якщо є операція або травма |
| Конкретні заборони | Текст | «не бігти», «пульс не вище 120», «не більше 20 хвилин» |
| Термін дії | Вибір | До дати / до дозволу лікаря / безстроково |
| Нотатки лікаря | Текст | Цитати і рекомендації із призначень |

> 🔒 Медичний розділ Олени не видно Сержу в жодному місці. Тільки загальний статус «режим відновлення» у спільній статистиці.

### Розділ В — Розклад і нагадування

| Поле | Тип | Опис |
|------|-----|------|
| Бажаний час | Вибір | Ранок / День / Вечір / По-різному |
| Кількість на тиждень | Вибір | 1–2 / 3–4 / 5+ / розберемось |
| Нагадування | Toggle + час | Push-нагадування. М'який тон. |
| Тижневий підсумок | Toggle + час | Неділя ввечері / понеділок вранці |

### Розділ Г — Система

| Поле | Опис |
|------|------|
| Ручне введення Garmin | Форма: кроки, середній пульс, макс. пульс |
| Режим звуку | Toggle муркотіння. За замовчуванням: вимкнено. |
| Часовий пояс | Для правильного відображення дат |
| Мова | Українська (за замовчуванням) |
| Видалити дані профілю | З підтвердженням. Чистить IndexedDB. |
| Про додаток | Версія PWA, умови, застереження безпеки |

**Обов'язкове застереження безпеки:**
> «Цей застосунок не є медичною консультацією. Якщо є гострий біль, різке погіршення стану або лікарські обмеження — тренування треба узгоджувати з лікарем або реабілітологом.»

## Зведена структура даних MVP

| Сутність | Основні поля |
|----------|-------------|
| **Profile** | id, name, mode, ageGroup, gender, activityLevel, goals, schedule, medical, partner |
| **Activity** | id, profileId, date, mode, duration, distance, steps, pace, avgHeartRate, notes |
| **CheckIn** | id, activityId, profileId, type, sleep, fatigue, pain, mood, readiness, difficulty, feeling, redFlags |
| **Recommendation** | date, profileId, targetDuration, paceType, cautionLevel, reason, kasperPhrase |
| **SafetyFlag** | id, profileId, date, level, symptom, source, action, resolved |
| **WeeklySummary** | weekId, profileId, stats, aiInsight, jointStats, nextFocus |

> Всі дані зберігаються локально (IndexedDB) у v0.1. Кожен запис має мітку `profileId` і `mode`.
